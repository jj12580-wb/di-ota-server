## 新问题

### 1. validatePasswordPolicy 成为死代码（Low）

**文件:** `backend/internal/server/router.go:903`

该函数已定义但不再被调用（本地登录改为 bcrypt 校验，不再需要密码策略）。保留会造成代码冗余。

### 2. Worker 的 t_task_stats INSERT 缺少去重/清理逻辑（Medium）

**文件:** `backend/cmd/worker/main.go:45-61`

每次周期执行都 `INSERT INTO t_task_stats`，没有 ON CONFLICT 也没有 DELETE 旧数据。随着运行时间增长，t_task_stats 会无限累积同一 task_id 的重复快照。

**建议:** 使用 INSERT ... ON CONFLICT (task_id) DO UPDATE（需唯一索引）或定期清理过期快照。

### 3. OIDC 回调未校验 state 参数（Medium）

**文件:** `backend/internal/server/router.go:134`

`/sso/callback` 接收 `code` 参数但忽略了 `state` 参数。OIDC 规范要求校验 state 以防止 CSRF 攻击。虽然当前为 MVP 阶段，但 `/sso/login` 生成了 state 并返回给前端，回调端应该验证其一致性。
