# OTA Server 评审答复

日期：2026-04-28

## 答复原则
- 本答复按三类处理：必须接受、部分接受（降级与改口径）、拒绝或延后。
- 只对有事实依据且与当前阶段目标一致的问题做立即改动。
- 对于缺少证据或仅属优化建议的问题，明确降级或进入后续迭代。

## 一、必须接受（已处理）

### 1) 硬编码本地登录凭证
处理结论：接受并修复。
处理说明：
- 移除硬编码 admin / Admin@123456 校验。
- 改为 LOCAL_AUTH_ENABLED 开关 + LOCAL_ADMIN_PASSWORD_HASH（bcrypt）校验。
- 默认关闭本地登录，推荐走 OIDC。
涉及文件：
- backend/internal/server/router.go
- backend/internal/config/config.go
- .env.example

### 2) 启动自动执行 Schema
处理结论：接受并修复。
处理说明：
- 增加 API_AUTO_MIGRATE_ON_START 开关。
- 默认 false，避免每次启动都跑建表 SQL。
涉及文件：
- backend/cmd/api/main.go
- backend/internal/config/config.go
- .env.example

### 3) 状态管理缺少数据库约束
处理结论：接受并修复。
处理说明：
- 为 t_release_task.state 增加 CHECK 约束。
- 为 t_package.status 增加 CHECK 约束。
- 使用约束存在性判断，兼容已创建表的场景。
涉及文件：
- backend/migrations/001_init.sql

### 4) Worker 仅心跳无业务
处理结论：接受并修复。
处理说明：
- 实现周期任务：写入 t_task_stats 快照。
- 实现失败率超阈值自动回滚 Running 任务。
涉及文件：
- backend/cmd/worker/main.go

### 5) 前端缺少开发代理
处理结论：接受并修复。
处理说明：
- 为 /api 与 /device 增加 Vite 代理到 ota-api:8080。
涉及文件：
- frontend/vite.config.ts

### 6) 时间戳 ID 生成
处理结论：接受并修复。
处理说明：
- 资源 ID 改为 UUID（package/task/trace）。
涉及文件：
- backend/internal/server/router.go

### 7) 正则重复编译
处理结论：接受并修复。
处理说明：

涉及文件：
- backend/internal/server/router.go

### 8) 断言错误无日志
处理结论：接受并修复。

## 三、拒绝或延后

### 12) “.env 文件提交仓库”按 Critical 处理
处理结论：拒绝该定级。
理由：
- 仅凭工作区存在 .env 不能证明已提交到版本库。
- 当前 .gitignore 已忽略 .env。
后续建议：
- 如需定级，请基于 git 历史提供证据。
涉及文件：
- .gitignore

### 13) 幂等先查后写
处理结论：延后。
理由：
- 当前通过 SQL ON CONFLICT 已保证正确性。
- 先查后写属于写放大优化，不是正确性缺陷。

### 14) 未使用表与未使用配置
处理结论：延后。
理由：
- 属于功能骨架/预留，不影响当前主链路可用性。
- 待下一阶段按模块建设时统一清理或接入。
