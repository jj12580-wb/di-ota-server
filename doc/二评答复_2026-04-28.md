## 二、新问题逐项答复

### 1) validatePasswordPolicy 未被调用
处理结论：问题成立，已整改。

处理说明：
- 评审指出“当前未被调用”属事实，属于代码可维护性问题。
- 已移除未使用的密码策略函数及对应预编译正则，减少无效代码与维护噪音。

### 2) Worker 向 t_task_stats 持续 INSERT 可能累积
处理结论：问题成立，已整改。

处理说明：
- t_task_stats 的当前定位是“周期快照/审计轨迹”，不是“每 task 一条当前态缓存”。
- 评审关于“会无限增长”的判断成立，当前实现缺少生命周期治理。
- 不采用 ON CONFLICT 覆盖（避免丢失时间序列），保留快照模型。
- 已在 Worker 周期任务中加入按时间窗口清理（可配置保留小时数）。
- 并增加 `t_task_stats(snapshot_time)` 索引，降低清理扫描成本。

### 3) OIDC 回调未校验 state
处理结论：问题成立，已整改。

处理说明：
- 评审关于 CSRF 防护缺失的结论成立，state 不校验不符合 OIDC 推荐实践。
- 已实现签名 state（HMAC-SHA256）方案，state 内含 nonce 与过期时间。
- `/sso/callback` 现已强制校验 state 格式、签名和有效期，校验失败直接拒绝请求。
- 实现为无状态验签，不引入会话存储依赖，符合当前架构边界。

## 三、结论
- 二评新提 3 项问题已在本轮全部完成整改。
- 我们坚持“避免不必要修改”，但对真实问题不拖延：确认即修复。
